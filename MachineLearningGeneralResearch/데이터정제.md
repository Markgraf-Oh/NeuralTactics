# 데이터 정제 (Data Preprocessing) 가이드

머신러닝 모델의 성능을 최적화하기 위해 게임 엔진에서 추출한 원시 데이터를 신경망이 이해하기 쉬운 형태로 변환하는 과정입니다.

---

## 1. 스케일링 (Scaling): 숫자 크기 맞추기

게임 엔진의 데이터는 단위가 제각각입니다 (예: 체력 1000, 쿨타임 5.5초, 언리얼 좌표 15400.0). 신경망은 숫자가 너무 크거나 단위가 다르면 혼란을 겪기 때문에, **0~1** 또는 **-1~1** 범위로 압축해야 합니다.

### Min-Max 정규화 (Min-Max Normalization)
가장 명확한 한계치(Max)가 있는 데이터에 사용합니다.
- **공식:** `(현재값 - 최소값) / (최대값 - 최소값)`
- **예시:** 현재 체력 300, 최대 체력 1000 → `0.3`으로 입력

### 표준화 (Standardization / Z-Score)
최대치가 명확하지 않거나 평균이 중요한 데이터에 사용합니다. (평균을 0, 표준편차를 1로 맞춤)
- **예시:** 에이전트가 받은 누적 데미지량

---

## 2. 범주형 데이터 인코딩 (Categorical Encoding): 상태를 숫자로 번역

게임의 '상태(State)' 데이터를 단순 숫자로 표현(예: 0: 대기, 1: 이동, 2: 공격, 3: 기절)하면, 신경망은 "기절(3)이 대기(0)보다 3배 더 크다"라는 잘못된 수학적 오해를 할 수 있습니다.

### 원-핫 인코딩 (One-Hot Encoding)
서로 독립적인 상태를 0과 1의 배열로 쪼개어 입력합니다.
- **예시:**
    - 대기 중: `[1, 0, 0, 0]`
    - 공격 중: `[0, 0, 1, 0]`
- **적용처:** 현재 장착 중인 무기 타입, 버프/디버프 종류, 지형의 종류 등

---

## 3. 좌표계 변환 (Coordinate Transformation): AI 중심의 세상

언리얼 엔진의 '월드 좌표(Absolute Position)'를 그대로 사용하면 AI는 특정 맵의 특정 위치에서만 싸울 수 있게 됩니다. 이를 해결하기 위해 좌표계를 변환합니다.

### 상대 좌표계 변환 (Relative Position)
모든 위치 데이터를 **Soldier AI(자기 자신)를 중심(0, 0, 0)**으로 변환합니다.
- **공식:** `적의 상대 위치 = 적의 월드 좌표 - 나의 월드 좌표`

### 로컬 좌표계 변환 (Local Space) 및 방향 정렬
AI가 바라보는 방향(Forward Vector)을 기준으로 (앞, 뒤, 좌, 우)로 회전시켜 입력하면 학습 속도가 훨씬 빨라집니다.

> **참고:** 현재 연구 중인 **EGNN(Equivariant GNN)**을 사용한다면, 모델 자체가 회전과 이동에 대한 불변성(Equivariance)을 보장하므로 회전 정렬 과정 없이 상대 좌표만 주어도 공간을 충분히 이해할 수 있습니다.

---

## 4. 결측치 및 예외 처리 (Handling Missing & Outliers)

### 더미 값(Dummy Value) 삽입 및 마스킹
정보가 없을 때(예: 시야에 적이 없을 때의 거리)를 처리하는 방법입니다.
- **방법 1:** 거리를 **최대 시야 거리**(정규화 시 1.0 이상)로 강제 설정
- **방법 2:** '적이 존재하는가?'를 나타내는 **원-핫 인코딩 플래그**(0 또는 1)를 추가하여, 플래그가 0일 때 거리값을 무시하도록 유도

### 클리핑 (Clipping)
물리 엔진 버그 등으로 발생하는 비정상적인 값(Outlier)이 신경망을 파괴하는 것을 방지합니다.
- **방법:** `Clamp(-1.0, 1.0)` 등을 사용하여 최대/최소값을 강제로 제한
